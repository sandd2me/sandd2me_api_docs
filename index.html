    Introduction

    The Sandd2me API is organized around REST. Our API has predictable, resource-oriented URLs, and uses HTTP response codes to indicate API errors. We use built-in HTTP features, like HTTP authentication and HTTP verbs, which are understood by off-the-shelf HTTP clients. JSON is returned by all API responses, including errors.

    We differentiate between a test and live environment. Domains will be shared with you upon becoming a Sandd2me business partner.

    You can view code examples in the dark area to the right.

    Authentication

    To authorize, use this code:

    # With shell, you can just pass the correct header with each request

$ curl https://api.sandd2me.nl/v1/endpoint \
-H "Authorization: Bearer AllThePackages"

    Authenticate your account when using the API by including your secret API key in the request. Your API key carries many privileges, so be sure to keep them secret! Do not share your secret API keys in publicly accessible areas such GitHub, client-side code, and so forth.

    Authentication to the API is perfomed via bearer auth token.

    All API requests must be made over HTTPS. Calls made over plain HTTP will fail. API requests without authentication will also fail.

    Make sure to replace ‘AllThePackages’ with your API key ✌️.

    This endpoint creates and order

    HTTP Request

    POST https://api.sandd2me.nl/v1/orders

    Query Parameters

    Parameter Required Format Description
    recipient_first_name true /^.+$/ The first name of the recipient of the package
    recipient_last_name true /^.+$/ The last name of the recipient of the package
    recipient_company_name false /^.+$/ The company name of the recipient
    recipient_street true /^.+$/ The street of the recipient of the package
    recipient_house_number true /^.+$/ The house number of the recipient of the package
    recipient_house_number_addition false /^.+$/ The house number addition of the recipient of the package
    recipient_postal_code true /^\d{4}\s?[A-z]{2}$/ The postal code of the recipient of the package
    recipient_city true /^.+$/ The city of the recipient of the package
    recipient_country_code true /^[A-Z]{2}$/ The country code of the recipient of the package using the ISO 3166 format
    recipient_email true /^.+$/ The email of the recipient
    recipient_phone false /^.+$/ The phone number of the recipient
    delivery_instructions false /^.+$/ Extra nstructions that the recipient might want to add
    reference false /^.+$/ Rhw order reference
    package_count true Integer The number packages
    weight true /^\d+$/ The weight of the package
    value_added_services false [String] Selection of value added services (see below)

    Time slots

    When you create an order, it will be placed in a timeslot. All orders that are created before 1.00pm(CET) will be placed in the timeslot between 6.00pm(CET) and 10.00pm(CET) on the same day. Orders that are created after 1.00pm(CET) will be placed in the timeslot 6.00pm(CET) and 10.00pm(CET) on the next day. The delivery slot is also found in the response that is given after an order is created.

    Errors

    The Sandd2me API uses the following error codes:

    Error Code Meaning
    400 Bad Request – Your request is just.. bad.
    401 Unauthorized – Your API key is wrong
    404 Not Found – The stuff you requested could not be found
    406 Not Acceptable – You requested a format that isn’t json 😱
    422 Unprocessable Entity – Used for validation errors
    429 Too Many Requests – You’re bombarding us with requests! Slow down!
    500 Internal Server Error – We had a problem with our server. Please try again later.
    503 Service Unavailable – We’re temporarily offline for maintenance. Please try again later.
